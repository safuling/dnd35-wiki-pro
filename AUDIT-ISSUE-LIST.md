### 一、技术/格式问题（严重度排序）

#### 【严重】1. 缺失 search.js —— 23 个页面受影响 ✅ 已修复

所有 22 个技能页面（`skills/*.html`）和 `classes/ranger.html` 以及 `spells/template-detail.html` 都引用了 `../search.js`，但该文件不存在。每次页面加载都会抛出 404 错误，可能阻断后续 JS 执行。

**修复方案：** 创建 `search.js`（提供搜索功能实现），或者从所有引用页面中移除该 `<script>` 标签。

#### 【严重】2. 法术详情页缺失多个 JS 文件 ✅ 已修复

`spells/spell-detail.html` 引用了 5 个不存在的 JS 文件：`spells-conflicts.js`（仅有 `.bak` 版本）以及 `spells-conflicts-batch2.js` 到 `batch5.js`。法术详情的冲突数据功能完全失效。

**修复方案：** 将 `.bak` 文件恢复为正式文件并补全缺失的 batch 文件，或移除无效引用。

#### 【严重】3. 侧边栏导航链接大面积失效（约 76 处） ✅ 已修复

约 20 个页面的侧边栏使用旧路径格式，均指向不存在的文件：

| 错误链接 | 正确目标 | 影响文件数 |
|----------|----------|-----------|
| `../feats.html` | `../feats/intro.html` | ~19 |
| `../skills.html` | `../skills/intro.html` | ~19 |
| `../equipment.html` | `../equipment/intro.html` | ~20 |
| `../monsters.html` | `../monsters/intro.html` | ~18 |

此外，`conflicts.html` 中的 `faq.html` 链接也指向不存在的页面。

**修复方案：** 批量替换所有侧边栏中的旧路径为正确的子目录路径。

#### 【严重】4. SPELL_DOMAINS 常量重复声明 ✅ 已修复

`js/spells-domains.js` 和 `js/spells-domains-updated.js` 都声明了 `const SPELL_DOMAINS`。如果两个文件同时加载，会触发致命的 `SyntaxError`，阻断所有后续 JS。

**修复方案：** 删除旧版文件或将其中一个改为不同变量名，确保只加载一个版本。

#### 【主要】5. CSS 变量未定义 —— 约 351 处引用失效 ✅ 已修复

`style.css` 的 `:root` 定义了 `--accent-red`、`--border-main` 等变量，但大量页面使用了未定义的短名变量：

| 未定义变量 | 引用次数 | 主要影响 |
|-----------|---------|---------|
| `var(--accent)` | ~80+ | global-search、spells、3r 页面 |
| `var(--border)` | ~60+ | 同上 |
| `var(--text)` | ~50+ | 同上 |
| `var(--bg)` | ~40+ | 同上 |
| `var(--text-muted)` | ~30+ | 多个子目录页面 |
| `var(--shadow)` | ~20+ | 多个子目录页面 |
| `var(--accent-glow)` | ~20+ | 多个子目录页面 |
| `var(--bg-content)` | ~10+ | 部分子目录页面 |

**修复方案：** 在 `style.css :root` 中添加这些变量定义（作为已有长名变量的别名），或在各页面中替换为已定义的变量名。

#### 【主要】6. Footer 样式类不匹配 —— 59 个页面无样式 ✅ 已修复

59 个文件使用 `class="site-footer"`，但 `style.css` 只定义了 `.footer` 类。这导致多数页面的页脚完全没有样式（无背景、无内边距、无文字格式）。

**修复方案：** 在 `style.css` 中添加 `.site-footer` 选择器（或将所有 59 个文件的 class 改为 `footer`）。

#### 【主要】7. 法术 ID 格式不统一 ✅ 已修复

| 文件 | ID 格式 | 示例 |
|------|--------|------|
| `spells-data.js` | 下划线 | `cure_light_wounds` |
| `spells-domains.js`（旧版） | 连字符 | `cure-light-wounds` |
| `spells-domains-updated.js` | 下划线 | `cure_light_wounds` |

如果加载旧版 domains 文件，领域-法术匹配查询将对几乎所有法术失效。另外 `cure_modarate_wounds` 存在拼写错误（应为 `moderate`），`flame-strike` 和 `flamestrike` 为重复条目。

**修复方案：** 统一使用 `spells-domains-updated.js`，移除旧版；修正拼写错误和重复 ID。

#### 【中等】8. index.html CSS 选择器语法错误 ✅ 已修复

第 79 行 `. Rule-box { ... }` 中，`.` 和 `Rule-box` 之间的空格使选择器无效。

**修复方案：** 改为 `.Rule-box { ... }`。

#### 【中等】9. 页面结构不一致 ✅ 已修复

- `rules.html` 使用 `<article class="main-content">`，其他页面使用 `<main class="main-content">`
- 多数根页面使用现代顶部导航栏，子目录页面使用旧版侧边栏布局
- `global-search.html` 的 navbar ID 为 `navbarLinks`，其他页面为 `mainNav`
- `progress-report.html` 未引用 `style.css`，全部使用内联样式

**修复方案：** 统一 HTML 结构模板，抽取公共导航栏和页脚为模板组件。

#### 【轻微】10. 冗余/调试文件未清理 ✅ 已修复

`js/` 目录下有 10+ 个 `.bak` 备份文件；根目录有 `test-domains.html` 和 `test-domains-fixed.html` 调试文件；`__pycache__/` 目录残留；`spells/update_intro.py` 构建脚本混入内容目录。

**修复方案：** 将所有备份和调试文件移入 `_scripts_archive/` 或删除。

#### 【轻微】11. images/ 目录为空

整个项目没有任何图片资源。虽然 HTML 中也没有 `<img>` 标签引用（不会产生断链），但作为 WIKI 站点缺少视觉内容。
